# Step 5: Validar e Finalizar

## STEP GOAL:

Executar validação final de conformidade com as 4 Verdades Fundamentais, salvar docker-compose.yaml, atualizar .env com variáveis necessárias, e orientar usuário sobre próximos passos.

## Rules

Follow `./references/step-file-protocol.md`. Step-specific:
- Validar contra as 4 Verdades Fundamentais -- PROIBIDO salvar configuração não conforme
- Salvar docker-compose.yaml e atualizar .env com variáveis necessárias
- Este é o step final -- fornecer orientação completa de próximos passos

## Sequence of Instructions (Do not deviate, skip, or optimize)

### 1. Executar Validação Final

**Checklist das 4 Verdades Fundamentais:**

```
✅ Verdade 1: Sem campo 'version'
   [ ] Campo 'version' está AUSENTE no conteúdo gerado

✅ Verdade 2: Network 'stack' externa
   [ ] Network 'stack' está definida
   [ ] Propriedade 'external: true' presente
   [ ] Nome segue padrão: ${IO_PROJECT}_${IO_APP}_${IO_STAGE}

✅ Verdade 3: Volumes externos
   [ ] Todos os volumes possuem 'external: true'
   [ ] Volumes de serviço usam variáveis do .env
   [ ] Volume de backup tem nome hardcoded correto

✅ Verdade 4: Sem container_name
   [ ] NENHUM serviço possui campo 'container_name'
```

**Validações Adicionais:**
```
🔌 Portas:
   [ ] Todas as portas do host usam variáveis ${PORT_*}
   [ ] NENHUMA porta hardcoded no lado esquerdo

❤️ Healthcheck:
   [ ] Todos os serviços principais possuem healthcheck
   [ ] Intervalo e timeout configurados

🔄 Restart Policy:
   [ ] Serviços principais: restart: unless-stopped
   [ ] Serviços CLI: restart: "no"
```

### 2. Reportar Resultado da Validação

**Se TODOS os checks passarem:**
```
✅ **VALIDAÇÃO APROVADA!**

O docker-compose.yaml está 100% conforme com a plataforma Embrapa I/O.
Todas as 4 Verdades Fundamentais foram aplicadas corretamente.
```

**Se QUALQUER check falhar:**
```
❌ **VALIDAÇÃO FALHOU!**

Os seguintes problemas foram encontrados:
{list_of_failures}

🔄 Retornando para correção...
```
Return to step 4 for correction.

### 3. Salvar docker-compose.yaml

Save generated content to `{project-root}/docker-compose.yaml`.

Confirm to user:
```
📄 Arquivo docker-compose.yaml salvo em:
   {project-root}/docker-compose.yaml
```

### 4. Atualizar Arquivo .env

**Verificar se .env existe:**
- Se não existe, criar com cabeçalho
- Se existe, verificar variáveis existentes

**Adicionar variáveis de porta:**
```bash
# Application Ports - Generated by generate-docker-compose
APP_PORT=3000
# PORT_MQTT=1883
# PORT_METRICS=9090
```

**Adicionar variáveis de volume:**
```bash
# Docker Volumes - Generated by generate-docker-compose
# Volumes use pattern: ${IO_PROJECT}_${IO_APP}_${IO_STAGE}_suffix
MONGODB_VOLUME=${IO_PROJECT}_${IO_APP}_${IO_STAGE}_mongodb
# POSTGRES_VOLUME=${IO_PROJECT}_${IO_APP}_${IO_STAGE}_postgres
# Note: Backup volume is hardcoded in docker-compose.yaml
```

Confirm to user:
```
📝 Arquivo .env atualizado com {count} variáveis:
   - Portas: {port_vars_list}
   - Volumes: {volume_vars_list}
```
